CallMe

Видеоконференции в браузере

Интро

CallMe разрабатывался как удобный и гибкий инструмент для организации внутренних видеоконференций по заказу компании Nvision Group. Прежде всего, система решает задачу модульной интеграции функционала видеоконференцсвязи в существующие и разрабатываемые проекты компании(преимущественно, заказы Ростелеком). Система работает без дополнительного ПО на клиентских машинах: только браузер.

Принцип работы

Система распределенная, пиринговая. Центральным узлом является сигналинг-сервер, предназначенный для передачи сообщений между клиентами и отправки управляющих команд. Видеопакеты через сервер не проходят, что позволяет минимизировать стоимость серверного оборудования. Система успешно работает на Raspberry PI. Серверна часть реализована на Erlang — платформа, ставшая стандартом надежности и отказоустойчивости в телекоммуникационных системах.

Интеграция

Система подразумевает интеграцию с сервером авторизации целевого продукта. При этом никаких ограничений на технологии авторизации нет — они настраиваются индивидуально под проект и под технологии, используемые в этом проекте.

Пользователь открывает страницу браузера с модулем CallMe, после чего управляющий сервер выполняет авторизацию и запрашивает публичные мета-данные о пользователях, например: имя, должность, департамент, фото и т.д. В минимальной имплементации требуется только имя пользователя для его идентификации в контакт-листе другими пользователями.

Технологические ограничения

Клиентская часть работает на WebRTC — p2p технологии, которая до сих пор находится в стадии активной разработки. В настоящий момент поддерживаются Chrome 21+, Firefox 12+, IE(6+). Для полноценной работы в IE требуется установить дополнительный модуль IE Chrome Frame — продукт компании Google для реализации функционала Chrome. При этом для пользователей визуально нет различий между IE6 и IE6 + Chrome Frame. Пакет очень прост в установке, и подгружается лишь при наличии управлящего мета-тега в коде разметки веб-страницы.

Что касается максимального количества участников, оно ограничено пропускной способностью сети клиентов. При этом канал адаптивный — между клиентами с плохой сетью будет передаваться видео более плохого качества, в то же врем между клиентами с широким каналом — хорошего качества.
